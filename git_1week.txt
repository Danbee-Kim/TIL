2. 1주차 주요 개념 키워드 적어보기

- Git:프로젝트 관리를 위한 도구

- Github:프로젝트 저장 공간을 제공

- commit:현재 프로젝트 상태를 저장한 것을 커밋이라고 표현

- 버전관리를 한다는 것은 프로젝트 상태가 변경되는 정보를 알고 있다는 것입니다. Git 은 commit 을 사용해서 버전이 달라지는 것을 관리합니다.

- 지금까지 우리가 한 작업은 'git 초기화하기(initialize) - add(staging) - commit' 입니다.

- repo : 'Git으로 관리되는 프로젝트' 를 Git 에서는 repo(리포, repository 리포지토리의 약자) 라고 부름

- push로 로컬 repo를 원격 repo로 보내고, pull로 원격에서 로컬로 받아오는 것 

- clone: 원격 repo를 내 컴퓨터에 가져와서 초기 repo세팅하는 것
